# 交接说明(给 Codex / 任何接手者)

项目:AI 主播直播数据统计后台,基于 RuoYi(backend=RuoYi-Vue 3.9.2,frontend=RuoYi-Vue3)。
需求详见同目录 `需求分析.md`。

## 当前状态

底座已跑通:前端 :80,后端 :8080,库 ry-vue,Redis Docker(ruoyi-redis),登录 admin/admin123,验证码已关。

「图片上传」模块**代码已写完**,分别提交在两个仓库的 `feature/live-upload` 分支(未推送、未测试):

- backend 新增:`com.ruoyi.live` 包(domain/mapper/service ×2 + XML ×2)、`web/controller/live/` 两个 Controller、`sql/live_role_menu.sql`(角色菜单)、`sql/live_tables.sql`(7 张业务表)、`sql/init-live.bat`(一键初始化,运行时输密码)
- frontend 新增:`src/api/live/upload.js`、`src/views/live/upload|review|stats/index.vue`(upload 是完整功能页,另两个是占位)

接口:POST `/live/upload/img`(批量截图)、POST `/live/upload/report`(汇报文本)、GET `/live/upload/list|daily`、DELETE、GET `/live/streamer/listAll`。纯 streamer 角色账号强制只能操作自己(见 LiveUploadController.getOwnStreamerIfRestricted)。图片存 `{ruoyi.profile}/live/{日期}/{主播ID}/`,profile 已配置为 J:/codex-projects/ai-streamer-stats/uploads。
