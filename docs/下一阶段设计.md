# AI 主播直播数据统计后台 - 下一阶段设计

版本:v0.2
日期:2026-07-03

## 1. 当前基线

已完成的第一版闭环是:

1. 主播/运营选择业务日期、主播
2. 上传打赏榜截图、聊天截图
3. 提交每日文字汇报
4. 查看按日完整性汇总
5. 查看上传明细、预览图片、删除记录

这一版解决的是“资料能不能收上来”。下一阶段要解决“资料怎么被 AI 识别、怎么被运营校正、怎么变成统计报表”。

## 2. 产品分工

### 2.1 主播手机端

主播只需要最短路径:

- 今日/昨日日期选择
- 上传打赏榜截图
- 上传聊天截图
- 填写工作汇报
- 查看自己是否已提交完整

手机端不展示复杂表格,全部使用卡片、标签和底部提交按钮。

### 2.2 运营电脑端

运营负责处理数据质量:

- 查看所有主播每日提交情况
- 批量触发 AI 识别
- 校正打赏榜识别结果
- 合并疑似重复客户
- 确认入库

电脑端保留表格、筛选、批量操作。

### 2.3 管理员/老板视角

管理员关注结果:

- 主播日/周/月 Xu 排名
- 主播提交完整率
- 客户打赏 Top
- 客户复购/连续出现
- 导出 Excel

## 3. 下一阶段开发顺序

### 阶段 A: 上传批次与 AI 识别记录

目标:让每次上传都能进入“待识别/识别中/待校正/已确认”的状态流。

需要补充:

- `live_upload_batch`:一次提交批次
- `live_ai_task`:AI 识别任务
- `live_ai_item`:AI 识别出的原始明细

原因:

- 现在 `live_upload` 是每张图一条记录,缺少“同一次提交”的批次概念
- 打赏榜多张截图需要作为一个批次一起识别,否则无法去重和判断排名连续性
- AI 原始结果不能直接写入客户/打赏正式表,必须先进入待校正区

建议状态:

| 状态 | 含义 |
|------|------|
| 0 | 待识别 |
| 1 | 识别中 |
| 2 | 待校正 |
| 3 | 已确认 |
| 4 | 识别失败 |

### 阶段 B: 识别校正页

目标:运营可以把 AI 原始识别结果修正成可靠数据。

页面结构:

- 左侧:上传图片预览
- 右侧:AI 识别明细表
- 顶部:主播、日期、类型、识别状态
- 底部:保存草稿、确认入库

打赏榜校正字段:

- 排名
- 客户昵称
- Xu
- 徽章
- 置信度
- 是否重复
- 关联客户

聊天截图校正字段:

- 客户昵称
- 互动日期
- 消息条数(可选)
- 关联客户

汇报文本校正字段:

- 主播
- 日期
- 总 Xu
- 原文

### 阶段 C: 客户库与合并

目标:客户可以沉淀,但不要被 AI 错误污染。

规则:

- AI 识别阶段只生成临时客户昵称
- 运营确认时才创建或关联 `live_customer`
- `live_customer` 不再强制昵称唯一
- 新增 `live_customer_alias` 保存历史昵称/别名
- 支持人工合并客户

客户合并后:

- 历史打赏记录归到主客户
- 保留别名映射
- 后续出现相同别名时提示疑似同一人

### 阶段 D: 数据统计页

目标:让数据能直接服务运营管理。

第一版统计:

- 今日提交完整率
- 主播日 Xu 排名
- 主播周/月 Xu 趋势
- 主播聊天跟进客户数
- 客户打赏 Top
- 识别待处理数量

筛选维度:

- 日期范围
- 主播
- 客户
- 识别状态

导出:

- 主播日汇总 Excel
- 打赏客户明细 Excel
- 待校正任务 Excel

## 5. API 草案

### 5.1 AI 任务

- `POST /live/ai/recognize/{batchId}` 触发识别
- `GET /live/ai/task/list` 识别任务列表
- `GET /live/ai/task/{taskId}` 识别详情
- `POST /live/ai/task/{taskId}/retry` 重试

### 5.2 校正

- `GET /live/review/list` 待校正列表
- `GET /live/review/{batchId}` 校正详情
- `PUT /live/review/item/{itemId}` 修改单条识别结果
- `POST /live/review/{batchId}/confirm` 确认入库

### 5.3 统计

- `GET /live/stats/overview` 看板概览
- `GET /live/stats/streamer-rank` 主播排名
- `GET /live/stats/customer-rank` 客户排名
- `GET /live/stats/trend` 趋势
- `POST /live/stats/export` 导出

## 6. AI 接入策略

第一步不要直接接真实 AI,先做 Mock AI:

1. 上传后生成任务
2. 点击“模拟识别”
3. 后端根据文件名或固定样例生成若干识别明细
4. 运营在校正页修改
5. 确认入库

Mock 跑通后再接真实视觉模型。这样能先验证业务闭环,避免一开始被模型调用、费用、图片格式拖住。

真实 AI 接入时要求:

- API key 只放后端配置
- prompt 版本入库
- 原始响应完整保存
- JSON 解析失败要留错误信息
- 低置信度字段在校正页高亮
